# Фаза 2. Распределение работ фазы 3

В этой фазе была решена задача **распределения работ по фазе 3** между участниками команды с учётом:

- оценок сложности задач в **Story Points (SP)**,
- индивидуальных **предпочтений** участников (насколько кому хочется делать ту или иную задачу),

## Итог распределения

В результате решения задачи распределения (с учётом ограничений по Story Points и предпочтений участников) задачи фазы 3 были разложены следующим образом:

- **Александр Елисеев (A)**:
  - **KAN13** — определить границы системы Auto Import Catalog и основных акторов;
  - **KAN14** — набросать и первично проработать не менее 10 Use Case (основные, альтернативные, негативные сценарии);
  - **KAN15** — провести совместную ревизию и уточнение Use Case (согласование внутри команды);
  - **KAN17** — рассчитать COSMIC Function Points проекта на основе выделенных функциональных процессов.

---

- **Вадим Ремезов (B)**:
  - **KAN16** — выделить функциональные процессы и движения данных для COSMIC FP;
  - **KAN18** — определить акторов и оценить их сложность (UAW в Use Case Points);
  - **KAN19** — оценить сложность Use Case (UUCW);
  - **KAN20** — оценить технические и средовые факторы и рассчитать итоговые Use Case Points и трудоёмкость.

---

- **Александр Павлов (C)**:
  - **KAN21** — провести коллективную экспертную оценку трудозатрат Use Case;
  - **KAN22** — агрегировать оценки (COSMIC, UCP, экспертные) в единую оценку проекта;
  - **KAN23** — проанализировать результаты: масштаб проекта, необходимый состав команды, сроки, фичи-кандидаты на отбрасывание.
