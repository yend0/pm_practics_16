# ФАЗА 5 — Планирование проекта (Auto Import Catalog)

## 1. Цель фазы

Целью фазы 5 является построение календарного плана выполнения проекта MVP **Auto Import Catalog** с учётом:
- декомпозиции проекта на задачи,
- ограничений по ресурсам (типам исполнителей),
- зависимостей предшествования,
- применения эвристических методов планирования.

Для получения плана использовался **Serial Schedule Generation Scheme (SSGS)** и набор классических эвристик упорядочивания работ.

---

## 2. Исходные данные

### 2.1 Состав команды и ресурсы

| Роль | Обозначение | Доступность |
|---|---|---|
| Backend-разработчик (Middle) | BE_M | 8 ч/день |
| Backend-разработчик (Junior) | BE_J | 8 ч/день |
| Frontend-разработчик (Junior) | FE_J | 8 ч/день |
| Тестировщик | QA | 8 ч/день |
| Проектный менеджер | PM | 8 ч/день |

Для каждой задачи в CSV-файле были заданы **фактические трудозатраты в часах по ролям**, которые затем переводились в нагрузку ресурсов в формате *часов в день*.

---

### 2.2 Входной план задач

Проект был декомпозирован на **22 задачи**, каждая из которых характеризуется:
- плановой длительностью (`Duration_days`),
- трудозатратами по ролям (часы),
- зависимостями предшествования (AND-зависимости).

Задачи связаны нелинейным графом с параллельным выполнением backend- и frontend-работ и точками синхронизации перед интеграцией и тестированием.

---

## 3. Метод планирования

Для построения календарного плана использовалась следующая схема:

1. Формирование **Activity List** (топологически корректного порядка задач).
2. Декодирование Activity List в календарное расписание с помощью **SSGS**, обеспечивающего:
   - выполнение всех предшествований,
   - соблюдение ограничений по ресурсам (часы/день).
3. Повторение процедуры для различных эвристик.
4. Сравнение результатов по **makespan** (общей длительности проекта).

---

## 4. Использованные эвристики

Были применены следующие эвристики упорядочивания работ:

- **SLK** — по возрастанию полного резерва
- **FREE** — по возрастанию свободного резерва
- **LST** — по возрастанию позднего начала
- **LFT** — по возрастанию позднего окончания
- **LSTLFT** — по возрастанию суммы LST и LFT
- **GRPW** — по убыванию длительности задачи и её последователей
- **LPT** — по убыванию длительности
- **MIS** — по убыванию числа последователей
- **GRD** — по убыванию произведения длительности и суммарных затрат ресурсов
- **GRWC** — по убыванию суммарных затрат ресурсов
- **GCRWC** — по убыванию затрат задачи и её последователей
- **ROT** — по убыванию отношения нагрузки к доступным ресурсам
- **RANDOM** — случайные Activity List (500 прогонов)

### 5.2 Лучшее решение

**Лучшая эвристика:** `SLK`  
**Makespan:** **55 рабочих дней**  
- ≈ **11 недель** (5-дневная рабочая неделя)
- ≈ **2.5 месяца** (22 рабочих дня в месяце)

Выбор эвристики SLK логичен, так как она приоритизирует задачи с минимальным резервом, эффективно сокращая критический путь проекта.
