# Use Case Points (UCP)

## Типы акторов в UCP

| Тип актора  | Описание                      | Вес |
| ----------- | ----------------------------- | --- |
| **Простой** | Другая система, API, cron     | 1   |
| **Средний** | Пользователь через простой UI | 2   |
| **Сложный** | Пользователь через сложный UI | 3   |

## Классификация Use Case’ов
| Тип Use Case | Кол-во транзакций | Вес |

## Technical Complexity Factor (TCF)

| № | Фактор | Оценка (0–5) | Комментарий |
|---|--------|-------------|-------------|
| T1 | Распределённая система | 3 | Веб-сервис + внешние источники |
| T2 | Требования к производительности | 3 | Массовые импорты и поиск |
| T3 | Эффективность использования ресурсов | 2 | Важна, но не критична |
| T4 | Сложная обработка данных | 4 | Парсинг, агрегация, сравнение |
| T5 | Повторное использование компонентов | 2 | Возможна, но не ключевая |
| T6 | Простота установки | 1 | Типовой веб-сервис |
| T7 | Простота использования | 3 | Ориентир на обычных пользователей |
| T8 | Кроссплатформенность | 1 | Веб |
| T9 | Лёгкость изменений | 3 | Частые изменения каталогов |
| T10 | Параллельность | 2 | Фоновые задачи |
| T11 | Безопасность | 3 | Доступ к коммерческим данным |
| T12 | Интеграции с внешними системами | 4 | Поставщики, сайты, API |
| T13 | Обучение пользователей | 1 | Минимальное |
|  | **ΣTF** | **32** |  |
