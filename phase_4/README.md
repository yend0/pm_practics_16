# ФАЗА 4 — Прогнозирование выполнения проекта (Метод Монте-Карло)

## Проект: Auto Import Catalog (A4)

---

## 1. Цель фазы

Целью фазы 4 является прогнозирование сроков выполнения **фазы 5 (Планирование)** с учётом неопределённости, используя **метод Монте-Карло**.  
Прогноз выполняется двумя независимыми моделями, основанными на истории выполнения задач в предыдущих фазах проекта.

---

## 2. Исходные данные

Для моделирования использовались данные из Jira-экспорта (CSV), содержащие следующие поля:

- **Start date** — фактическая дата начала задачи  
- **Срок исполнения** — фактическая дата завершения задачи  
- **Затраченное время** — фактические трудозатраты по задаче (в часах)

Период анализа: **10 сентября 2025 — 24 ноября 2025**.  
История содержит периоды активной работы и недели без выполнения задач, что важно для корректного моделирования.

---

## 3. Декомпозиция фазы 5

| № | Задача | Оптимистично (дни) | Наиболее вероятно (дни) | Пессимистично (дни) |
|---|--------|--------------------|--------------------------|---------------------|
| 1 | Формирование структуры проекта и списка работ | 0.25 | 0.75 | 1.5 |
| 2 | Определение ролей и доступных ресурсов | 0.10 | 0.30 | 0.75 |
| 3 | Оценка длительности и ресурсных требований задач | 0.50 | 1.50 | 3.0 |
| 4 | Определение зависимостей и ограничений | 0.50 | 1.25 | 2.5 |
| 5 | Настройка и запуск алгоритмов планирования | 0.50 | 1.25 | 2.5 |
| 6 | Анализ результатов и формирование итогового плана | 0.25 | 0.75 | 1.5 |

---

## 4. Модель 1 — Прогноз на основе количества задач в неделю

### 4.1 Идея модели

- Из истории проекта вычисляется **количество задач, завершённых в каждую неделю**.
- Это значение рассматривается как **дискретное распределение случайной величины**.
- В каждой итерации симуляции:
  - случайно выбирается, сколько задач выполняется за неделю;
  - это значение вычитается из общего числа задач фазы 5;
  - подсчитывается количество недель до завершения всех задач.

### 4.2 Результаты

- Получено распределение длительности выполнения фазы 5 в неделях.
- В качестве итоговой оценки используется **P80** — срок, в который фаза 5 завершается с вероятностью 80%.

## 5. Модель 2 — Прогноз на основе трудозатрат (часы в неделю)

### 5.1 Идея модели

- Из CSV-файла формируется распределение **доступных человеко-часов в неделю**:
  - значение поля *«Затраченное время»* распределяется равномерно по интервалу выполнения задачи;
  - далее часы агрегируются по неделям.
- Для каждой задачи фазы 5 длительность сэмплируется из **треугольного распределения** (O, M, P).
- Суммарная трудоёмкость фазы 5 (в часах) уменьшается каждую неделю на случайно выбранную доступную ёмкость команды.

### 5.2 Результаты

- Получено распределение длительности выполнения фазы 5 в неделях.
- Итоговая оценка также определяется по **P80**.

## 6. Сравнение моделей

- **Модель 1** проста и хорошо отражает общую скорость выполнения задач, но чувствительна к малому числу задач.
- **Модель 2** более реалистична, так как учитывает трудоёмкость задач и реальную доступность команды.
- Различие между результатами моделей объясняется:
  - неоднородностью задач фазы 5;
  - наличием недель без активности;
  - вариативностью трудозатрат.

---

## 7. Итоговый вывод

Использование двух моделей Монте-Карло позволяет получить более надёжный прогноз сроков выполнения фазы 5.  
Результаты моделей согласуются между собой по порядку величин, а различия объясняются разной степенью детализации входных данных.

Для планирования рекомендуется ориентироваться на **оценку P80 второй модели**, так как она учитывает фактические трудозатраты и лучше отражает реальные условия выполнения проекта.
